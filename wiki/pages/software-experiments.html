All vs Subset of Crosslinks

We run our pipeline by choosing random subsets of crosslinks. We estimate a solution to the localization problem to built the supporting framework for structures involved, which is non-convex by nature. We vary the starting seed to generate multiple supporting framework. The final solution, after registration, is therefore multiple conformations. Note, that each iteration takes around 1-1.5 minutes to compute. Thus the user can quickly geenrate multiple structures based on the crosslink entries selected.

We also tried by choosing all crosslinks, but it resulted in clashes. This is expected since the magtinude of distance between the C-alpha atoms, obtained from experimental crosslink data vary due to the flexibility of the side-chain and the backbone chain of the protein. We hypothesize that a fraction of accurate crosslinks lead to an accurate structure. We test our algorithm by choosing random subsets of crosslinks. Despite using only a fraction of the crosslinks, the end result satisfied a large percentage of the entire set of crosslinks. This confirms that IMPROVised can be used for efficient and accurate modeling o fthe three dimensional structrues for the protein complexes.

Evaluation

We get different solutions to localization problem for different runs, which results in different registered structures. We evaluated these structures based on percentage of correctly satisfied crosslinks.

Recall that a crosslink is defined a distance of 30 angstroms between a pair of C-alpha atoms. In reality they refer to the distance between the corresponding amino acid residues. Thus a tolerance value of 5 angstrom is added to the magnitude of crosslink while evaluating for violations. Once the final structures are modeled, the distance between every pair of C-alpha atoms are taken which constitute the crosslinks. If the distance is more than 35 angstrom, then it is noted as a violation.
